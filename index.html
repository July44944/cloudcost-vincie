CSV format guide

This tool ships with parsers for four simplified cloud billing export formats, plus a generic template. The format is auto-detected from the header row on upload — no need to pick one manually.

FieldDescriptionRequired
dateBilling date, format YYYY-MM-DDRequired
provideraws / azure / gcp / huaweiRequired
accountAccount / subscription / project nameRequired
serviceService name, e.g. EC2, Storage, Cloud SQLRequired
regionResource regionOptional
resource_idUnique resource identifierRequired
team / project / envCost-attribution tagsOptional — missing values count toward untagged spend
pricing_modelon-demand / reserved / committed / spotOptional
usage_qty / usage_unitUsage quantity and unitOptional
costLine cost amount (number)Required
currencyCurrency, e.g. USDOptional, defaults to USD
utilization_pctAverage utilization 0–100, used for idle/oversized detectionOptional
attachedWhether a storage volume is attached, true / falseOptional

Mirrors the core fields of a real AWS Cost & Usage Report (CUR), trimmed down. Field names are kept identical so you can drop in a real export directly.

pricing/termOnDemand / Reserved

Mirrors the core fields of a real Huawei Cloud "Resource Details" bill export — modeled on an actual export's column structure (with fictional resource and tenant data). Two things make this format different from the other three: it's monthly grain (one row per resource per Billing Cycle, not per day), and the payment column name carries the settlement currency — e.g. Payment Amount(AED) — which gets converted to USD on import using a fixed FX rate so it can be summed alongside other providers.

Huawei fieldMaps to
Billing CycleBilling date (e.g. "Jan 2023" → first of month)
Account NameAccount
Service Type NameService name
Region NameRegion
Resource ID / Resource Name / Offering Instance IDResource ID (first non-empty of the three)
Enterprise Project NameCost tag — a non-"default" project counts as tagged (Huawei has no separate team tag in this export)
Billing ModeOn Demand / Yearly-Monthly
Payment Amount(<currency>)Cost amount — currency read from the column name itself
Avg CPU UtilizationAverage utilization (custom extension field)
Volume AttachedVolume attachment state (custom extension field)
